程式小白的 GitHub 入門

GitHub
是什麼？

不需要會寫程式，也能理解並使用 GitHub。用最直覺的方式帶你認識這個工具。

用四個類比來理解

☁️

第一個概念

雲端硬碟

就像 Google Drive，你的檔案存在網路上，任何裝置、任何地方都能存取，不怕電腦壞掉或遺失。

存放內容
⏪

第二個概念

時光機

每次修改都會被記錄，隨時可以回到過去任何一個版本。就像 Google 文件的版本紀錄，但功能更強大。

版本控制
🌐

第三個概念

免費網站主機

把你的筆記或 HTML 檔案直接變成一個有網址的公開網站，不需要租伺服器，完全免費。

GitHub Pages
👥

第四個概念

協作空間

多人可以同時在同一個專案上工作，每個人的修改都被追蹤記錄，不會互相覆蓋或衝突。

團隊合作
GitHub 就像一本有自動存檔、可以協作、
還能對外公開的筆記本。

一次操作的完整流程

從寫一個字，到變成網站

每次你在 GitHub 上做的事，背後其實就是這四個步驟。

Step 01

建立 Repository（資料夾）

每個專案或筆記本都放在一個 repo 裡。就像在桌面新增一個資料夾，但這個資料夾存在雲端。

類比：在 Google Drive 新增資料夾

Step 02

編輯內容

直接在網頁上編輯 README.md，用 Markdown 語法寫出有格式的筆記。不需要安裝任何軟體。

類比：在 Google Docs 打字

Step 03

Commit（存檔）

按下「Commit changes」，這次修改就被永久記錄下來。每個 commit 都有時間戳記，可隨時回溯。

類比：按下「儲存」，但可以無限回溯

Step 04

GitHub Pages 自動發布

開啟 Pages 功能後，每次 commit 都會自動更新你的網站。約 1～2 分鐘後即可看到新版本。

類比：存檔後網站自動同步更新

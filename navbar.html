Code

瀏覽所有檔案、編輯內容、查看每次 commit 的修改歷史。進 repo 第一個看到的地方。

最常用
◎

Issues

開待辦票、回報問題、追蹤任務進度。可以加標籤、指派負責人、設截止日期。

所有人
⇄

Pull requests

在 branch 上改好後，提交一份「申請合併」讓別人審核，確認沒問題才合進 main。

協作用
▶

Actions

查看自動化流程的執行紀錄。每次 commit 後的自動部署、測試結果都在這裡顯示。

常用
▦

Projects

看板式任務管理，可以建立「待辦 / 進行中 / 完成」欄位，就像輕量版的 Trello。

進階用
📖

Wiki

撰寫專案的說明文件，像一本內部百科全書。適合放詳細的操作說明或背景資料。

團隊用
🛡

Security

檢查 repo 裡有沒有安全漏洞或意外洩露的密碼、API 金鑰等敏感資訊。

開發者
📈

Insights

查看 repo 的統計數據：有多少人看過、貢獻者活動、commit 頻率、流量來源。

進階用
⚙

Settings

管理 repo 的一切設定：開啟 GitHub Pages、設定權限、改名稱、或刪除 repo。

管理者
